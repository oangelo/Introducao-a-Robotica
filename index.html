Projeto de Robótica

Dinâmica e Expectativas

Visão Geral do Projeto

  • Trabalho em grupos de 3
  • Pesquisa e apresentação sobre um tema de robótica
  • Desenvolvimento colaborativo de critérios de avaliação (rubrica)
  • Apresentações em reveal.js (30 min + 10 min para perguntas)
  • Avaliação por pares e reflexão
  • Todo o processo documentado no GitHub

Divisão do Tópico e Rotação de Papéis

  • Divida o tópico principal em pelo menos 3 subtemas
  • Cada membro assumirá um papel diferente para cada subtema
  • Garante que todos experimentem cada papel
  • Promove uma compreensão abrangente do projeto

Papéis no Grupo

Cada membro assumirá um papel específico para cada subtema, rotacionando entre os subtemas:

  • Pesquisador: Coleta e organiza informações relevantes
  • Desenvolvedor: Cria o conteúdo da apresentação em reveal.js
  • Revisor: Verifica qualidade, coerência e faz edições finais

Responsabilidades dos Papéis

  • Pesquisador: Busca fontes confiáveis, resume informações, propõe estrutura do conteúdo para o subtema
  • Desenvolvedor: Transforma a pesquisa em slides reveal.js, cria gráficos/diagramas, formata o conteúdo do subtema
  • Revisor: Verifica precisão, clareza, coerência, sugere melhorias, faz correções finais para o subtema

Uso do GitHub

  • Todo o conteúdo do projeto deve estar no GitHub
  • Crie uma branch para cada subtema
  • Faça commits frequentes para registrar seu progresso
  • Use pull requests para revisões e merges
  • Use issues para rastrear tarefas e discussões
  • Mantenha um documento de planejamento mostrando a divisão de subtemas e papéis

O que é uma Rubrica?

  • Ferramenta de avaliação que lista critérios específicos
  • Define diferentes níveis de desempenho para cada critério
  • Ajuda a entender as expectativas do projeto
  • Guia tanto o desenvolvimento quanto a avaliação do trabalho

Desenvolvimento da Rubrica

  • Sessão de brainstorming em sala
  • Discussão e refinamento dos critérios
  • Definição dos níveis de desempenho para cada critério
  • Finalização colaborativa da rubrica
  • A rubrica será usada para auto-avaliação e avaliação por pares

Formato da Apresentação

  • Usando reveal.js (como esta apresentação)
  • 30 minutos de apresentação
  • 10 minutos para perguntas e discussão
  • Deve incluir elementos visuais e, se possível, demonstrações práticas
  • Todos os membros do grupo devem participar

Avaliação por Pares

  • Use a rubrica desenvolvida em sala
  • Seja construtivo e respeitoso
  • Forneça comentários específicos e acionáveis
  • Avalie todas as apresentações, exceto a do seu próprio grupo
  • Submeta avaliações via GitHub como issues

Etapas do Projeto

  1. Formação de grupos, escolha do tema, divisão em subtemas, setup do GitHub
  2. Desenvolvimento da rubrica e início da pesquisa nos subtemas
  3. Pesquisa e desenvolvimento do conteúdo em reveal.js para cada subtema
  4. Integração dos subtemas, revisão final e ensaios
  5. Apresentações e avaliações por pares
  6. Reflexão final e discussão em grupo

Expectativas

  • Participação ativa de todos os membros do grupo em cada etapa
  • Pesquisa aprofundada e conteúdo original para cada subtema
  • Apresentações envolventes e informativas
  • Respeito e apoio mútuo durante todo o processo
  • Abertura para feedback e disposição para aprender
  • Uso efetivo do GitHub para documentação e colaboração
